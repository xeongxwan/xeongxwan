# Hey, there

I prefer writing code that solves real problems.  
Not just code that works, but code that stays understandable and maintainable over time.

I value simple designs with clear responsibilities over unnecessary complexity.  
Practical decisions matter more to me than excessive patterns or abstractions.

Tools and technologies come after thinking and structure.  
I aim for code that is easy to read, improve, and trust.
